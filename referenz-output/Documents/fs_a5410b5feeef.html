Schaltflächen (Artefaktpaket-Assistent)

Symbol

Schaltfläche

Beschreibung

„Speichern & Schließen“

[Strg] + [H]

Speichert die Änderungen im Artefaktpaket-Assistenten als Change bzw. Change Set und schließt den Dialog.

Beachten Sie, dass eine eventuell bereits gespeicherte Paketdatei [Link] zum Change/Change Set nicht automatisch gelöscht bzw. aktualisiert wird. Wenn Sie Änderungen an der Artefakt-Zusammenstellung vorgenommen haben, müssen Sie die Paketerstellung erneut ausführen, damit das Paket die Änderungen enthält.

Verfügbarkeit: Nur im erweiterten Modus

„Speichern“

[Strg] + [S]

Speichert die Änderungen im Artefaktpaket-Assistenten als Change bzw. Change Set.

Beachten Sie, dass eine eventuell bereits gespeicherte Paketdatei [Link] zum Change/Change Set nicht automatisch gelöscht bzw. aktualisiert wird. Wenn Sie Änderungen an der Artefakt-Zusammenstellung vorgenommen haben, müssen Sie die Paketerstellung erneut ausführen, damit das Paket die Änderungen enthält.

Verfügbarkeit: Nur im erweiterten Modus

„Löschen“

[Strg] + [D]

Löscht den Change/das Change Set nach der Bestätigung einer Sicherheitsabfrage dauerhaft aus der Datenbank des TRMS.

Auf IKAROS-Systeme, in denen der Change/das Change Set bereits installiert wurde, hat das Löschen keinen Einfluss.

Wenn ein Change oder Change Set in einem Transportauftrag zugewiesen ist, wird das Löschen verhindert. Changes können zudem nicht gelöscht werden, wenn sie in einem Change Set enthalten sind.

Verfügbarkeit: Nur im erweiterten Modus

„Quellsystem auswählen“

[Strg] + [Q]

Öffnet den Dialog zum Anmelden an ein System, um das IKAROS-System, aus dem die Artefakte gewählt werden sollen, neu festzulegen.

In bereits gespeicherten Changes oder Change Sets kann das Quellsystem nicht mehr geändert werden, weshalb die Schaltfläche dann deaktiviert ist.

Verfügbar: in allen Modi

„In anderes TRMS transportieren“

[Strg] + [Alt] + [i]

Öffnet zunächst einen Dialog zur Auswahl der Zielsysteme und startet anschließend die Übertragung des Changes/des Change Sets in die Ziel-TRMS-Systeme (wo sie als schreibgeschützte Changes/Change Sets angelegt werden [Link]).

„Transportieren“

[Strg] + [i]

Leitet den Transport eines Artefaktpakets in ein oder mehrere IKAROS-Systeme ein.

  In Changes/Change Sets: Öffnet zunächst einen Dialog zur Auswahl der Zielsysteme und beginnt dann mit dem Transport und der Installation des Changes/Change Sets in diesen Systemen. Wenn mehrere Zielsysteme ausgewählt sind, erfolgt der Transport sequenziell nacheinander und für jedes Zielsystem wird ein eigenes Protokoll geschrieben.

  Im Menüpunkt „Pakete“: Öffnet zunächst einen Dialog zum Hochladen eines vorhandenen „.trms4“-Pakets und anschließend den Assistenten zum Transportieren und Installieren der Inhalte aus dem Paket in ein oder mehrere Zielsysteme [Link]. Wenn mehrere Zielsysteme ausgewählt sind, erfolgt der Transport sequenziell nacheinander und für jedes Zielsystem wird ein eigenes Protokoll geschrieben.

Das Protokoll hierzu können Sie im Menüpunkt „Transportprotokolle“ öffnen.

Verfügbarkeit: in allen Modi

„Erstellen“

[Strg] + [P]

Startet die Erstellung der Paketdatei und leitet am Ende den Download im Webbrowser ein. Der Artefaktpaket-Assistent bleibt weiterhin geöffnet.

  In Changes/Change Sets: Die Paketdatei wird immer auch binär in der Datenbank gespeichert; das lokale Herunterladen ist optional.

  Im Menüpunkt „Pakete“: Sie müssen die Datei speichern, andernfalls geht sie verloren (sie wird nicht in der Datenbank gespeichert).

Verfügbar: in allen Modi

„Herunterladen“

[Strg] + [Umschalt] + [E]

Startet den Download des „.trms4“-Pakets zum Change/Change Set. Dies ist nur möglich, wenn zuvor ein Paket erstellt wurde.

„Markierungen“

  „Gemäß Filter setzen“

[Strg] + [Alt] + [A]

  „Gemäß Filter entfernen“

[Strg] + [Alt] + [B]

  „Alle in Liste entfernen“

[Strg] + [Alt] + [C]

  „Listenübergreifend entfernen“

[Strg] + [Alt] + [D]

Bei Klick wir die Schaltfläche aufgeklappt und besitzt drei verschiedene Optionen:

  „Gemäß Filter setzen“: Setzt in der geöffneten Listenansicht bei allen Artefakten, die den aktuell eingestellten Filterkriterien entsprechen, das „Ausgewählt“-Häkchen.

  „Gemäß Filter entfernen“: Entfernt bei den gefilterten Artefakten das „Ausgewählt“-Häkchen.

  „Alle in Liste entfernen“: Entfernt bei allen Artefakten desselben Typs das „Ausgewählt“ Häkchen (ungeachtet des aktuellen Filters).

  „Listenübergreifend entfernen“: Entfernt nach der Bestätigung einer Sicherheitsabfrage bei allen Artefakten im gesamten Paket (über alle Gruppen und Arten hinweg) das „Ausgewählt“-Häkchen sowie die Transportvorgaben,  sodass im gesamten Paket anschließend kein Artefakt mehr angehakt ist.

Beachten Sie, dass die Filterung nicht nur die sichtbaren Einträge betrifft, sondern auch solche, die erst beim Scrollen in der Liste nachgeladen werden.

Verfügbar: in allen Modi

„Alles aufklappen“

Öffnet in der Baumansicht der Artefakte alle Knoten, sodass jeder Knoten sichtbar ist.

Verfügbar: in allen Modi (ausgeblendet in Change Sets, wenn Registerkarte „Change“ im Vordergrund)

„Alles einklappen“

Schließt in der Baumansicht der Artefakte alle Knoten, sodass nur noch die Gruppenknoten angezeigt werden.

Verfügbar: in allen Modi (ausgeblendet in Change Sets, wenn Registerkarte „Change“ im Vordergrund)

„Übersicht anzeigen“

[Strg] + [U]

Öffnet den Dialog mit einer Übersicht aller aktuell gewählten Artefakte über alle Gruppen und Artefakt-Typen hinweg [Link].

Zudem können Sie hier für jedes gewählte Artefakt die Transportvorgaben prüfen und bearbeiten.

Verfügbar: in allen Modi

„Ersetzen aus Datei“

Erlaubt das Ersetzen eines Artefakts in der Paketdatei mithilfe der beiden dazugehörigen XML-Dateien, die Sie im Menüpunkt [Transportsystem -> Artefakte] für das Artefakt exportieren können.

Die Schaltfläche ist immer aktiv, sobald im Change/Change Set eine Paketdatei gespeichert ist.

Unterschiede bestimmter Artefakt-Typen

  Alle außer SQL-Skripten:

-     Nach dem Betätigen der Schaltfläche erscheinen nacheinander zwei Datei-Hochlade-Dialoge. Im ersten müssen Sie die Datei mit dem Artefakt selbst hochladen, im zweiten die dazugehörige „-Content.xml“-Datei.

-     Durch das Ersetzen werden sowohl das Artefakt selbst als auch dessen Transportvorgaben in der Paketdatei ersetzt.

  SQL-Skripte:

-     Nach dem Betätigen der Datei erscheint ein Datei-Hochlade-Dialog, wo Sie die „.sql“-Datei auswählen müssen. Eine „-Content.xml“-Datei ist für diesen Artefakt-Typ nicht erforderlich.

-     Wichtig dabei: Die hochgeladene Datei und die zu ersetzende Datei müssen denselben Dateinamen haben.

Es ist nicht erforderlich, das gewünschte Artefakt im Paketassistenten vorher zu markieren, sondern das TRMS ermittelt es automatisch anhand der hochgeladenen Informationen.

Das zu ersetzende Artefakt muss im Change/Change Set bereits das „Ausgewählt“-Häkchen besitzen, andernfalls erhalten Sie nach dem Ersetzungsversuch die Fehlermeldung, dass das Artefakt gar nicht im Paket enthalten ist. Dieser Fehler tritt auch auf, wenn das Artefakt zwar nachträglich als „Ausgewählt“ angehakt wurde, die Paketdatei jedoch veraltet ist und das Artefakt deshalb noch nicht enthält.

Verfügbarkeit: Nur im erweiterten Modus

„Ersetzen aus Quellsystem“

Erlaubt das Aktualisieren eines Artefakts in der gespeicherten Paketdatei durch die aktuellste Version aus dem Quellsystem.

Die Schaltfläche ist nur aktiviert, wenn eine Paketdatei im Change/Change Set gespeichert ist und zudem momentan ein als „Ausgewählt“ markiertes Artefakt selektiert ist.

Falls das selektierte Artefakt trotz „Ausgewählt“-Häkchen nicht in der gespeicherten Paketdatei enthalten sein sollte, erhalten Sie eine entsprechende Fehlermeldung.

Durch das Ersetzen wird nur das Artefakt selbst in der Paketdatei ausgetauscht; die Transportvorgaben hingegen bleiben unverändert.

Hinweis: Workflow-Pakete und SQL-Skripte können nur per „Ersetzen aus Datei“ ersetzt werden.

Verfügbarkeit: Nur im erweiterten Modus

„Transportvorgaben“

[Strg] + [E]

Öffnet für das markierte Artefakt einen Dialog zum Einstellen der Transportvorgaben. Mit diesen kann das Verhalten sowohl für die Paketgenerierung (z. B. automatisches Integrieren abhängiger Datensätze) als auch die spätere Installation (überschreiben, falls schon vorhanden?) festgelegt werden.

Die möglichen Vorgaben unterscheiden sich je nach Typ des Artefakts.

Verfügbar: in allen Modi, wenn weder „Datenbankobjekt“ noch „Datenbankskript“ als Artefakt-Typ ausgewählt sind.

„Hinzufügen“

[Strg] + [Alt] + [A]

Öffnet einen Dateiauswahldialog, um ein Workflow-Aktualisierungspaket  (nur „.wfu4“-Dateien) hochzuladen und unverändert in das Paket zu integrieren.

Verfügbar: in allen Modi, wenn die Gruppe „Workflow-Aktualisierungs-Pakete“ ausgewählt ist.

„Reihenfolge“

[Strg] + [Alt] + [R]

 

Öffnet einen Dialog, in welchem Sie die Installationsreihenfolge für SQL-Skripte, Datenbankobjekte bzw. Workflow-Artefakte festlegen können. Dies kann erforderlich sein, falls Artefakte in einer bestimmten Reihenfolge in das Zielsystem eingespielt werden müssen.

Verfügbar: in allen Modi, wenn einer der Knoten „Datenmodell“, „Datenbankobjekt“, „Datenbankskript“ oder „Workflow-Aktualisierungs-Paket“ ausgewählt ist.

Registerkarte „Transportkontrollskripte“

Im Artefaktpaket-Assistenten in den Menüpunkten „Pakete“, „Changes“ und „Change Sets“ finden Sie im Menüband die Registerkarte „Transportkontrollskripte“ [Link] mit jeweils drei Schaltflächen pro Skript:

  Skript anzeigen (Code schreibgeschützt anzeigen, falls vorhanden)

  Skript bearbeiten (Code bearbeiten bzw. anlegen)

  Skript löschen (Skript aus dem Paket/Change/Change Set entfernen)

„Bearbeiten“ (Prerequisite)

[Strg] + [Umschalt] + [E]

Öffnet einen Dialog, in dem Sie den Code des jeweiligen Skripts einsehen und bearbeiten können.

Wenn das Skript gelöscht bzw. noch nie bearbeitet wurde, öffnet sich der Dialog mit leerem Code-Editor.

  Die Bearbeitung des Skript-Codes ist auch möglich, wenn noch keine Artefakte ausgewählt wurden.

  Falls die Paketdatei eines Changes/Change Sets bereits in der Datenbank vorliegt, wird dort automatisch die neueste Skriptversion hinterlegt, ohne dass die Paketdatei neu erzeugt werden muss.

[Strg] + [Umschalt] + [I]

„Bearbeiten“ (Finalisierung)

[Strg] + [Umschalt] + [F]

„Löschen“ (Prerequisite)

[Strg] + [Umschalt] + [D]

Löscht den Code des Skripts, sodass es nicht mehr in das trms4-Paket aufgenommen wird.

  Changes und Change Sets müssen Sie zusätzlich noch speichern, damit das Löschen des Skriptcodes auch endgültig in der Datenbank erfolgt.

  Falls die Paketdatei eines Changes/Change Sets bereits in der Datenbank vorliegt, wird dort automatisch die neueste Skriptversion hinterlegt, ohne dass die Paketdatei neu erzeugt werden muss.

Registerkarte „Dokumentation“

Auf der Registerkarte Dokumentation finden Sie Schaltflächen zum Hinzufügen einer Datei als kundenspezifischen Dokumentation für den Change/das Change Set.

Hinweis: Die generelle Hilfe/Dokumentation zum TRMS an sich können Sie durch Klick auf das Fragezeichen-Symbol rechts oben im Dialog aufrufen.

Symbol

Schaltfläche

Beschreibung

„Hinzufügen“

Öffnet einen Dateiauswahldialog, mit dem Sie dem Change/Change Set ein separates Dokument anhängen können. Hiermit können Sie sowohl für sich selbst Informationen über die Änderungen dokumentieren als auch eventuell gesetzlichen Dokumentationspflichten nachkommen.

Hinweis: Die Dokumentation befindet sich nur in der Datenbank des TRMS und wird nicht in der Paketdatei gespeichert bzw. transportiert.

Verfügbarkeit: Nur im erweiterten Modus

„Herunterladen“

Lädt die Dokumentationsdatei lokal auf den Rechner herunter.

Es ist nicht möglich, das Dokument zur Bearbeitung zu öffnen. Sie können es jedoch herunterladen, ändern und dem Change/Change Set erneut hinzufügen, um die vorherige Version zu ersetzen.

Hinweis: Die Dokumentation befindet sich nur in der Datenbank des TRMS und wird nicht in der Paketdatei gespeichert bzw. transportiert.

Verfügbarkeit: Nur im erweiterten Modus
