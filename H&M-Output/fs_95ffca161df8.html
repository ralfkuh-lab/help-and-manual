Hintergrund

Der Artefaktpaket-Assistent im TRMS ist eine Oberfläche, mit der Sie Zusammenstellungen von Artefakten festlegen und im TRMS als Change bzw. Change Set speichern oder als Paketdatei exportieren können.

Zudem erlaubt der Assistent auch den Transport inkl. der Installation dieser Artefakte in eines oder mehrere Zielsysteme.

Beim Aufruf des Artefaktpaket-Assistenten müssen Sie sich zunächst am gewünschten Quellsystem anmelden.

Die drei Modi des Artefaktpaket-Assistenten

Der Artefaktpaket-Assistent besitzt vier verschiedene Modi für verschiedene Anwendungsfälle:

Anwendungsfall

Menüpunkt

Besonderheiten

Einzelne Artefakte exportieren

„Artefakte“

•Artefakte bestehen aus zwei zusammengehörigen Einzeldateien („sql“, „wfu4“ oder „xml“ zzgl. „...-Content.xml“)

•Es kann keine Paketdatei erstellt werden.

•Es können nur Artefakte ausgewählt werden (also keine Changes/Change Sets)

•Paketdateien erstellen und speichern

•Paketdateien in ein IKAROS- oder TRMS-Zielsystem transportieren

•Paketdateien als Change/Change Set in das aktuelle TRMS importieren

„Pakete“

•Das erstellte Paket muss zum Abschluss heruntergeladen werden, weil die Daten sonst nirgends gespeichert werden.

•Es können nur Artefakte ausgewählt werden (also keine Changes/Change Sets)

•Paketzusammenstellung als „Change“ speichern,

•Paketdateien erstellen

„Changes“

•Die Paketkonfigurationen können als „Change“ in der Datenbank gespeichert werden.

•Die Artefakte aus einem Change können als Paketdatei exportiert oder direkt in Zielsystemen installiert werden.

•Es können nur Artefakte ausgewählt werden (also keine anderen Changes oder Change Sets)

•Paketzusammenstellung als „Change Sets“ speichern,

•Paketdateien erstellen

„Change Sets“

Dasselbe wie unter „Change“. Zusätzlich zu Artefakten können jedoch auch ganze Changes ins Paket aufgenommen werden.

Je nach Modus ist der Artefaktpaket-Assistent entsprechend um Schaltflächen, Zusatzinformationen für Changes/Change Sets bzw. die Registerkarte „Change“ erweitert.
