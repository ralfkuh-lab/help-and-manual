Automatisierung des TRMS mit dem Kommandozeilenprogramm „TrmsCli.exe“

Mittels der Konsolenanwendung „TrmsCli.exe“ können Sie lokal gespeicherte „.trms4-“-Paketdateien, die mit dem TRMS lokal exportiert wurden, automatisiert in ein IKAROS-Zielsystem transportieren und somit dort installieren.

Hinweis: Das Transportieren von Changes bzw. Change Sets direkt aus der TRMS-Datenbank ist hiermit nicht möglich. Sie könnten jedoch die Paketdateien für die Changes/Change Sets im TRMS erstellen, herunterladen und sie so lokal verfügbar machen.

Das Programm besitzt keine eigene Oberfläche, sondern muss z. B. per Kommandozeile oder PowerShell unter Angabe bestimmter Parameter ausgeführt werden.

Bereitstellung der Programmdateien

Die Anwendung muss nicht installiert werden, sondern Sie können sie aus einem beliebigen Ordner heraus nutzen. Eine Anbindung an das TRMS ist nur erforderlich, wenn das Protokoll zu dem Transportprozess im TRMS selbst gespeichert werden soll (Parameter „/LoggingInTRMS“) – anstatt als Datei (Parameter „TransportLogOutputPath“).

Die ausführbaren Dateien der Konsolenanwendung können im TRMS-Menüpunkt [Administration -> Dateizentrum] heruntergeladen werden.

Aufruf von Funktionalität

Dem Programm müssen Sie beim Aufruf per Parameter „/Function“ übergeben, welche der folgenden vier Funktionen ausgeführt werden soll:

•„Import“,

•„ImportSingleArtefact“

•„ShowCustomizingArtefactSources“,

•„EncryptPassword“ und

•„CheckWebserviceAccessibility“.

Return Codes der TrmsCli.exe

Die Konsolenanwendung „TrmsCli.exe“ kann die folgenden Return Codes zurückliefern:

Return Code

Bedeutung

Beschreibung

0

Erfolgreich

Die Funktionalität wurde erfolgreich ausgeführt.

10

Warnung

Der Transportprozess lief durch, jedoch wurde mindestens ein Artefakt (eventuell auch alle) aus fachlichen oder technischen Gründen nicht im Zielsystem installiert.

20

Fehler

IKAROS hat bei Ausführung der Funktionen einen Fehler gemeldet (z. B. bei inhaltlichen Prüfungen oder Datenkonstellationen).

30

Fehler

Die Ausführung der Funktionalität wurde wegen eines technischen Fehlers vorzeitig abgebrochen.

Hilfe-Parameter „/?“

Mittels des Hilfe-Parameters „/?“ können Sie sich die Funktionen und Parameter direkt in der Kommandozeile auflisten lassen.

•Abfragen aller Funktionen (Konsolenaufruf, CMD):

TrmsCli.exe /?

•Abfragen der Parameter zu einer bestimmten Funktion (Konsolenaufruf, CMD):

TrmsCli.exe /Function=Import /?

•Beispiel (Konsolenaufruf, CMD):

TrmsCli.exe /Function=Import /ImportFilePath="c:\tmp\dpSourceSetUps.trms4" ^
/TargetSystemStsAddress="http://localhost:2014/trust.svc" ^
/TargetSystemServiceHostAddress="http://localhost:2309" ^
/LoggingInTrms=true /TrmsStsAddress="http://localhost:2014/trust.svc" ^
/TrmsServiceHostAddress="http://localhost:4358"

Hinweise aus der Praxis mit der TrmsCli.exe

•Die TrmsCli.exe kann nicht aus Laufwerken ausgeführt werden, bei denen es sich lediglich um mit dem Befehl „SUBST“ gemappte lokale Verzeichnisse handelt.

•Auf dem Rechner, auf dem die „TrmsCIi.exe“ abgelegt und genutzt werden soll, muss das „Token Signing Certificate“ des Servers/Rechners installiert sein, auf dem der STS läuft, da andernfalls keine Kommunikation mit dem IKAROS-Zielsystem stattfinden kann. Meistens ist das der Server/Rechner, auf dem auch der IKAROS-Web-Server „Talos“ läuft.
